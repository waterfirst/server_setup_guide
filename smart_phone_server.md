# 📱 삼성 갤럭시 노트10 스마트폰 서버 구축 및 트러블슈팅 가이드

본 문서는 삼성 갤럭시 노트10(안드로이드 OS)을 활용하여 정적 HTML 웹 페이지 배포 및 학습용 미니 서버를 구축하는 전 과정과 실전 운영 중 발생한 트러블슈팅 내역을 정리한 품질팀 실전 가이드라인입니다.

---

## 1. 하드웨어 및 OS 공장초기화 (서버 최적화)
안정적인 백그라운드 리소스 확보를 위해 단말기를 완전히 제로 상태로 만드는 단계입니다.

* **기기 전체 초기화:** `설정 -> 일반 -> 초기화 -> 기기 전체 초기화` 진행
* **초기 세팅 필수 행동:**
  * **구글 및 삼성 계정 로그인 건너뛰기:** 백그라운드 동기화 및 자동 업데이트로 인한 리소스 낭비 방지
  * **화면 잠금 해제:** `설정 -> 잠금화면 -> 화면 잠금 방식 -> 설정 안 함` (재부팅 시 프로세스 차단 방지)

## 2. 서버 상시 구동을 위한 환경 설정 (안전장치)
안드로이드의 OS 차원의 백그라운드 프로세스 강제 종료(Sleep Mode)를 방지합니다.

* **개발자 옵션 활성화 및 화면 켜짐 유지:**
  1. `설정 -> 휴대전화 정보 -> 소프트웨어 정보 -> 빌드번호` 연속 7번 터치
  2. `설정 -> 개발자 옵션 -> 충전 중 화면 켜짐 상태 유지` 활성화 (상시 전원 공급 시 잠자기 진입 방지)
* **배터리 절전 예외 설정:**
  1. `설정 -> 배터리 및 디바이스 케어 -> 배터리 -> 앱별 배터리 사용 관리`에서 자동 절전 기능 일체 비활성화
  2. Termux 앱 속성 정보에서 **배터리 제한 없음(Unrestricted)**으로 설정
* **하드웨어 보호:** 상시 충전 시 배터리 팽창(스웰링) 및 발열 위험이 있으므로 `배터리 보호(85% 제한)` 기능을 켜고 통풍이 잘되는 곳에 배치합니다.

## 3. 네트워크 고정 IP 설정
공유기의 DHCP 스케줄에 의해 IP가 변경되어 접속이 끊기는 것을 방지합니다.

1. `설정 -> 연결 -> Wi-Fi` 이동 후 연결된 Wi-Fi 옆 ⚙️(톱니바퀴) 터치
2. **개인정보 보호:** `임의 MAC` ➡️ `휴대전화 MAC`으로 변경
3. **IP 설정:** `DHCP` ➡️ `고정(Static)`으로 변경
4. 현재 할당받은 단말기 IP 주소(예: `192.168.0.43`)를 확인 후 저장

## 4. Termux 환경 구축 및 서버 실행

### ① 패키지 다운로드 및 리눅스 환경 구축
* 구글 플레이 스토어 버전은 업데이트가 중단되었으므로 반드시 **F-Droid** 제공 패키지를 사용합니다.
* Termux 앱 실행 후 다음 명령어를 차례대로 실행합니다.


# 패키지 시스템 최신화 (Y/N 질의 시 Y 입력)
pkg update && pkg upgrade -y

# 기본 인프라 패키지(Python, OpenSSH, Nginx) 설치
pkg install python openssh nginx -y

# 백그라운드 수면 방지 웨이크록 적용
termux-wake-lock



# 웹 서버 홈 디렉토리 생성 및 이동
mkdir -p ~/site
cd ~/site

# 안전한 단선형 echo 명령어로 메인 웹 페이지 생성
echo "<html><body><h1>Galaxy Note10 Server Working!</h1><p>Quality demo page success!</p></body></html>" > index.html

# Python 웹 서버 구동 (포트 8000 지정)
python -m http.server 8000

## 🛠️ 실전 트러블슈팅 (Troubleshooting)

### 이슈 1. 특정 포트(예: 8000)로 웹 브라우저 접속이 안 되는 현상

* **원인 점검:**
1. 접속 기기(PC/타 단말)와 서버 스마트폰의 Wi-Fi SSID 불일치 (2.4GHz / 5GHz 분리 포함)
2. 서버 구동 명령어의 포트 지정 번호 불일치 (`http.server` 기본값은 8080)
3. 최신 웹 브라우저의 내부 검증 로직으로 인한 `https://` 자동 전환 오류


* **해결 행동:**
* 접속 기기가 동일 와이파이에 연결되었는지 확인하고, 주소창에 `http://[스마트폰IP]:8000` 형태로 명확하게 프로토콜 입력
* Termux에서 `ip route get 1.1.1.1 | awk '{print $7}'` 명령어를 활용하여 실제 활성화된 IP 주소를 정확하게 교차 검증



### 이슈 2. 리눅스 표준 IP 확인 명령어 미작동 (`no command ipconfig`)

* **원인 점검:** `ipconfig`는 Windows 전용 명령어로, Linux(Termux) 커널 환경에서는 지원하지 않음
* **해결 행동:** * 리눅스 표준 명령어인 `ifconfig` 입력 후 `wlan0` 인터페이스 항목 필터링
* 또는 가장 확실하게 IP 주소 한 줄만 출력하는 `ip route get 1.1.1.1 | awk '{print $7}'` 명령어 활용



### 이슈 3. 멀티 라인 파일 생성 문법(`cat << 'EOF'`) 사용 중 터미널 입력 무한 루프 버그

* **원인 점검:** 스마트폰 가상 키보드의 클립보드 연속 입력 오작동 및 `EOF` 종결 문자 처리 미숙으로 인해 터미널 세션이 닫히지 않고 지속 도배되는 현상 발생
* **해결 행동:**
1. **물리 리셋:** 스마트폰 상단 알림창(노티바)을 내려 Termux 진행 알림의 **`EXIT`** 버튼을 강제 터치하여 오염된 프로세스를 완전히 파괴(Kill)
2. **세션 초기화:** Termux 앱을 재시작하여 깨끗한 초기 명령어 입력 창(`$`) 재확보
3. **안전 문법 대체:** 백도어 입력 유도가 없는 안전한 단선형 스트림 명령어 구조로 대체하여 즉시 생성 완료
```bash
cd ~/site && echo "Galaxy Note10 Server Working!" > index.html

```
